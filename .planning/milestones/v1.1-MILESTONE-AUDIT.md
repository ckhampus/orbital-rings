## E2E Flows (3/3)

### Flow 1: Wish Fulfillment Chain
`WishFulfilled` → `HappinessManager.OnWishFulfilled()` → `_lifetimeHappiness++` + `MoodSystem.OnWishFulfilled()` → if tier changed: `EmitMoodTierChanged` + `EconomyManager.SetMoodTier()` → `MoodHUD` updates wish count (pulse) and tier label (cross-fade) → SaveManager autosave triggers

**Status:** COMPLETE

### Flow 2: Save/Load Cycle
`CollectGameState()` writes v2 fields → `ApplyState()` version-gates to v2 path → `RestoreState(lifetime, mood, baseline, ...)` → `_moodSystem.RestoreState()` recalculates tier → `EconomyManager.SetMoodTier(_lastReportedTier)` → `MoodHUD._Ready()` reads restored values

**Status:** COMPLETE

### Flow 3: Fresh New Game
`_lifetimeHappiness = 0` default → `_lastReportedTier = Quiet` default → `_currentTierMultiplier = 1.0f` default → `MoodHUD._Ready()` reads 0 wishes, Quiet tier → displays "♥ 0" + "Quiet"

**Status:** COMPLETE

## Milestone Definition of Done

From ROADMAP.md: *"Replace the single happiness percentage with a dual-value system (Lifetime Happiness + Station Mood) so the core loop stays alive indefinitely."*

**Achieved.** The old single-float happiness is completely replaced:
- Lifetime happiness (integer, never decreases) tracks permanent progress
- Station mood (float, decays toward rising baseline) provides dynamic gameplay feel
- Five named tiers govern arrival rate and income multiplier
- Save/load preserves both values with v1 backward compatibility
- HUD displays both values with animated feedback
- The deprecated float-space APIs (HappinessBar, Happiness shim, HappinessChanged event, SetHappiness) are fully removed
