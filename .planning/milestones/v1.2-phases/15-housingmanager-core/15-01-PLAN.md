  <action>
Replace the Phase 14 skeleton with the full assignment engine. Keep the existing namespace, class declaration, Instance property, Config export, and _EnterTree.

**Data structures (private fields):**
- `_housingRoomCapacities`: `Dictionary<int, int>` mapping anchor index to capacity (mirrors HappinessManager pattern)
- `_roomOccupants`: `Dictionary<int, List<string>>` mapping anchor index to list of citizen names assigned there
- `_citizenHomes`: `Dictionary<string, int>` mapping citizen name to home anchor index (reverse lookup)
- `_isRestoring`: `bool` flag to suppress event emission during save restoration

**Public API:**

`public static int ComputeCapacity(RoomDefinition definition, int segmentCount)` -- Returns `definition.BaseCapacity + (segmentCount - 1)`. This is the SOLE authority for capacity calculation per user decision.

`public int? GetHomeForCitizen(string citizenName)` -- Returns the anchor index of the citizen's home, or null if unhoused. Used by SaveManager and future UI.

`public int GetOccupantCount(int anchorIndex)` -- Returns the number of citizens assigned to the room at anchorIndex. Used internally and useful for future UI.

`public IReadOnlyList<string> GetOccupants(int anchorIndex)` -- Returns the list of citizen names assigned to a room. Used by future tooltip UI.

`public int TotalCapacity` -- Property returning sum of all `_housingRoomCapacities` values.

`public int TotalHoused` -- Property returning `_citizenHomes.Count`.

`public void RestoreFromSave(IReadOnlyList<(string citizenName, int? homeIndex)> assignments)` -- Rebuild internal state from save data without emitting events. Set `_isRestoring = true`, iterate assignments, for each non-null homeIndex that exists in `_housingRoomCapacities`, call the internal assign method. After all restorations, attempt to assign any remaining unhoused citizens. Set `_isRestoring = false`. Also set each CitizenNode's HomeSegmentIndex. See Pattern 4 in RESEARCH.md.

**Lifecycle:**

`_Ready()` -- Subscribe to GameEvents: RoomPlaced, RoomDemolished, CitizenArrived. Store delegate references for clean unsubscription. Call `InitializeExistingRooms()` to scan for pre-placed housing rooms (same pattern as HappinessManager.InitializeHousingCapacity).

`_ExitTree()` -- Unsubscribe all events. Clear Instance.

**Event handlers:**

`OnRoomPlaced(string roomType, int segmentIndex)` -- Query `BuildManager.Instance.GetPlacedRoom(segmentIndex)`. If null or not Housing category, return. Compute capacity via `ComputeCapacity(def, segmentCount)` using the new SegmentCount field. Cache in `_housingRoomCapacities[anchorIndex]`. Initialize `_roomOccupants[anchorIndex]` as empty list. Then call `AssignUnhousedCitizens(anchorIndex)` to batch-assign oldest-first unhoused citizens up to capacity. Per user decision: batch assignment in same frame, no staggered delays.

`OnRoomDemolished(int segmentIndex)` -- Look up anchor from `_housingRoomCapacities`. The room is already gone, so use the pre-cached dictionary (same pitfall as HappinessManager). Get the list of displaced citizens from `_roomOccupants[anchorIndex]`. Remove the room from both dictionaries. For each displaced citizen (oldest-first per user decision): remove from `_citizenHomes`, set their CitizenNode.HomeSegmentIndex to null, emit CitizenUnhoused event, then attempt reassignment via `FindBestRoom()`. Partial reassignment is allowed -- citizens that cannot be reassigned remain unhoused (no penalty per HOME-05).

`OnCitizenArrived(string citizenName)` -- Find best room via `FindBestRoom()`. If found, assign citizen. If no rooms available, citizen remains unhoused (no penalty).

**Internal helpers:**

`FindBestRoom()` -- Fewest-occupants-first with random tiebreak. Iterate `_housingRoomCapacities`, for each room with vacancy (occupants < capacity), track the one with fewest occupants. Break ties using reservoir sampling: `GD.Randi() % tieCount == 0`. Return anchor index or -1 if all full. See Pattern 3 in RESEARCH.md.

`AssignCitizen(string citizenName, int anchorIndex)` -- Add citizen to `_roomOccupants[anchorIndex]`, set `_citizenHomes[citizenName] = anchorIndex`, find the CitizenNode via CitizenManager.Instance.Citizens and set its `HomeSegmentIndex = anchorIndex`. If `!_isRestoring`, emit `GameEvents.Instance.EmitCitizenAssignedHome(citizenName, anchorIndex)` and `GD.Print($"Housing: {citizenName} assigned to room at segment {anchorIndex}")`.

`AssignUnhousedCitizens(int anchorIndex)` -- Iterate `CitizenManager.Instance.Citizens` in list order (oldest-first per user decision). For each citizen not in `_citizenHomes`, if `_roomOccupants[anchorIndex].Count < _housingRoomCapacities[anchorIndex]`, call `AssignCitizen`. Stop when room is full.

`AssignAllUnhoused()` -- For each unhoused citizen (oldest-first), call `FindBestRoom()` and assign if a room is found. Used after save restoration.

`InitializeExistingRooms()` -- Scan all 24 segment indices (`SegmentGrid.TotalSegments`). For each placed housing room (by anchor index, avoid double-counting), compute and cache capacity. Initialize empty occupant lists. Then attempt to assign any existing unhoused citizens. This handles the case where rooms exist before HousingManager subscribes.

**Silent engine per user decision:** No floating text, no sounds, no visual feedback. Events fire for downstream subscribers. Include `GD.Print` statements for assignment/displacement to aid manual verification.

**Parallel capacity tracking per user decision:** HousingManager uses size-scaled capacity (`BaseCapacity + segments - 1`). HappinessManager continues using flat `BaseCapacity`. Both track independently. Phase 16 will consolidate.
