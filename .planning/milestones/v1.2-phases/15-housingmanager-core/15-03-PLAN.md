<objective>
Fix stale home references on save/load by populating _housingRoomCapacities before RestoreFromSave checks it.

Purpose: UAT test 7 revealed that ALL citizens become unhoused on load because _housingRoomCapacities is empty when RestoreFromSave runs. Two population paths are both bypassed during save restoration: (1) InitializeExistingRooms is skipped because StateLoaded=true, (2) OnRoomPlaced never fires because BuildManager.RestorePlacedRoom does not emit the RoomPlaced event. The fix is to call InitializeExistingRooms() at the start of RestoreFromSave so the dictionary is populated from already-restored BuildManager rooms before any assignment lookups occur.

Output: Fixed HousingManager.cs that correctly restores housing state from saves.
</objective>
