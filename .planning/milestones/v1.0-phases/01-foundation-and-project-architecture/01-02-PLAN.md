**OrbitalCamera.cs** (`Scripts/Camera/OrbitalCamera.cs`):
- Namespace: `OrbitalRings.Camera`
- `public partial class OrbitalCamera : Node3D` (does NOT extend SafeNode — the camera is a scene-level node managing its own lifecycle, not a signal consumer)
- Scene structure assumption: This script attaches to a Node3D ("CameraRig") with a child Camera3D node

**Exported properties (all tunable in Inspector):**
- Orbit: `OrbitSpeed` (0.005f), `OrbitMomentumDecay` (0.92f), `KeyboardOrbitSpeed` (2.0f)
- Zoom: `ZoomMin` (5.0f), `ZoomMax` (25.0f), `ZoomSpeed` (1.5f), `ZoomSmoothing` (8.0f)
- Idle: `IdleOrbitSpeed` (0.02f), `IdleTimeout` (5.0f)
- Tilt: `TiltAngleDeg` (60.0f)

**Private state:**
- `_camera` (Camera3D child reference)
- `_orbitVelocity` (float, current rotational speed)
- `_targetZoom` / `_currentZoom` (float, zoom lerp pair)
- `_idleTimer` (float, seconds since last input)
- `_isDragging` (bool, right-mouse-button held)

**_Ready():**
- Get Camera3D child: `_camera = GetNode<Camera3D>("Camera3D")`
- Initialize zoom to ZoomMax (default view = whole ring visible, per user decision)
- Set camera rotation: `_camera.RotationDegrees = new Vector3(-TiltAngleDeg, 0, 0)`
- Set camera position: `_camera.Position = new Vector3(0, 0, _currentZoom)`

**_UnhandledInput(InputEvent @event):**
- Right-click press/release: toggle `_isDragging`
- Right-click drag (InputEventMouseMotion while dragging): set `_orbitVelocity = -mm.Relative.X * OrbitSpeed`, reset idle timer
- Scroll wheel up: `_targetZoom = Mathf.Max(_targetZoom - ZoomSpeed, ZoomMin)`, reset idle timer
- Scroll wheel down: `_targetZoom = Mathf.Min(_targetZoom + ZoomSpeed, ZoomMax)`, reset idle timer

**_Process(double delta):**
- Read keyboard orbit input: `float keyInput = Input.GetAxis("orbit_left", "orbit_right")`
  - If significant (abs > 0.01): add to orbit velocity, reset idle timer
- Apply orbit: `RotateY(_orbitVelocity)` (rotates the pivot Node3D on Y axis)
- Decay momentum: `_orbitVelocity *= OrbitMomentumDecay` (per user decision: cinematic glide after input stops)
- Clamp tiny velocities to zero: if `Mathf.Abs(_orbitVelocity) < 0.0001f` then `_orbitVelocity = 0f`
- Smooth zoom: `_currentZoom = Mathf.Lerp(_currentZoom, _targetZoom, ZoomSmoothing * (float)delta)`
- Update camera position: `_camera.Position = new Vector3(0, 0, _currentZoom)`
- Idle orbit: increment timer, if `_idleTimer > IdleTimeout && _orbitVelocity == 0f` then `RotateY(IdleOrbitSpeed * (float)delta)` (per user decision: gentle slow idle orbit when not interacting)
- Emit GameEvents: when orbit starts (velocity transitions from ~0 to significant) call `GameEvents.Instance?.EmitCameraOrbitStarted()`, when orbit stops call `GameEvents.Instance?.EmitCameraOrbitStopped()`. Use a `_wasOrbiting` bool to detect transitions. Use null-conditional `?.` since GameEvents might not be ready in edge cases.

**Input map in project.godot:**
Add `[input]` section with `orbit_left` and `orbit_right` actions. Rather than using the verbose Object() serialization format (which is fragile), add the input actions programmatically in OrbitalCamera._Ready() as a fallback:
```csharp
// Ensure input actions exist (in case project.godot entries are missing)
if (!InputMap.HasAction("orbit_left"))
{
    InputMap.AddAction("orbit_left");
    var keyA = new InputEventKey();
    keyA.PhysicalKeycode = Key.A;
    InputMap.ActionAddEvent("orbit_left", keyA);
    var keyLeft = new InputEventKey();
    keyLeft.PhysicalKeycode = Key.Left;
    InputMap.ActionAddEvent("orbit_left", keyLeft);
}
// Same for orbit_right with D and Right arrow
```
This is more reliable than editing project.godot's complex input serialization format. The input actions are still available to Input.GetAxis().

**Do NOT:**
- Use `_PhysicsProcess` for camera (it's purely visual, not physics-driven — pitfall #4 from research)
- Use SpringArm3D (unnecessary for fixed-tilt strategy camera — research alternatives considered)
- Auto-center on click (per user decision: camera stays put when clicking)

├── PlaceholderRing (Node3D)      # Ring parent at world origin

Material for the ring: Create an inline StandardMaterial3D in the .tscn with a warm pastel albedo color (e.g., Color(0.85, 0.75, 0.85) — soft lavender).

Placeholder boxes: Two small BoxMesh instances sitting on top of the ring surface at different positions, with different warm pastel colors, to provide scale reference and visual confirmation the scene is working.
