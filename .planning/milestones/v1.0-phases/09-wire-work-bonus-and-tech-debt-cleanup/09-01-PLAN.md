**CitizenNode.cs:**
1. Line 480: Change `EmitCitizenEnteredRoom(citizenName)` to `EmitCitizenEnteredRoom(citizenName, _visitTargetSegment)`. The `_visitTargetSegment` field (line 124) is set at line 397 before StartVisit begins, and the tween captures `citizenName` as a local (line 425). The field value is stable during the tween sequence.
2. Line 491: Change `EmitCitizenExitedRoom(citizenName)` to `EmitCitizenExitedRoom(citizenName, _visitTargetSegment)`. Note: `_visitTargetSegment` is reset to -1 at line 531 in Phase 8 callback, which runs AFTER Phase 5 where the exit emit happens. So the value is still valid at line 491.

**EconomyManager.cs:**
1. Add a `HashSet<string> _workingCitizens = new()` private field alongside existing `_workingCitizenCount` (line 43). This tracks which citizens are currently counted as working, solving the race condition where a room is demolished mid-visit (Research Pitfall 2).
2. In `_Ready()` (after line 77, the CreditsChanged emit), add event subscriptions with null guard following codebase pattern (see HappinessManager._Ready lines 169-174):
   ```csharp
   if (GameEvents.Instance != null)
   {
       GameEvents.Instance.CitizenEnteredRoom += OnCitizenEnteredRoom;
       GameEvents.Instance.CitizenExitedRoom += OnCitizenExitedRoom;
   }
   ```
3. Add `_ExitTree()` override (EconomyManager currently has none) that unsubscribes:
   ```csharp
   public override void _ExitTree()
   {
       if (GameEvents.Instance != null)
       {
           GameEvents.Instance.CitizenEnteredRoom -= OnCitizenEnteredRoom;
           GameEvents.Instance.CitizenExitedRoom -= OnCitizenExitedRoom;
       }
   }
   ```
4. Add private handler `OnCitizenEnteredRoom(string citizenName, int flatSegmentIndex)`:
   - Call `IsWorkRoom(flatSegmentIndex)` -- if true, add citizenName to `_workingCitizens` HashSet, set `_workingCitizenCount = _workingCitizens.Count`.
5. Add private handler `OnCitizenExitedRoom(string citizenName, int flatSegmentIndex)`:
   - If `_workingCitizens.Remove(citizenName)` returns true, set `_workingCitizenCount = _workingCitizens.Count`. This handles the demolished-room race condition: we check the citizen set, not the room existence.
6. Add private helper `IsWorkRoom(int flatSegmentIndex)`:
   ```csharp
   private bool IsWorkRoom(int flatSegmentIndex)
   {
       var room = Build.BuildManager.Instance?.GetPlacedRoom(flatSegmentIndex);
       return room?.Definition.Category == Data.RoomDefinition.RoomCategory.Work;
   }
   ```
7. In `RestoreCredits()` method (line 253), also clear the `_workingCitizens` HashSet and reset `_workingCitizenCount` to 0. This handles scene transitions (title screen to game) where stale entries would accumulate. RestoreCredits is called during save load, which is the same path where ClearCitizens is called.
