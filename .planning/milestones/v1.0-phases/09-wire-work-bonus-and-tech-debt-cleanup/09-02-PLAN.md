Two files, minimal changes:

**SafeNode.cs:**
1. In `_EnterTree()` (line 31): Add `base._EnterTree();` as the FIRST line, before `SubscribeEvents()`. Base calls go first in enter/construction, ensuring parent initialization happens before child work.
2. In `_ExitTree()` (line 36): Add `base._ExitTree();` as the LAST line, after `UnsubscribeEvents()`. Base calls go last in exit/destruction, ensuring child cleanup happens before parent teardown.

Result:
```csharp
public override void _EnterTree()
{
    base._EnterTree();
    SubscribeEvents();
}

public override void _ExitTree()
{
    UnsubscribeEvents();
    base._ExitTree();
}
```

This is a defensive fix. `Node._EnterTree()` is currently empty in Godot 4, but calling base ensures safety for: (a) future Godot versions adding behavior, and (b) deeper inheritance chains where an intermediate class adds logic.

**WishBoard.cs:**
1. In `SubscribeEvents()` (line 173): Add `if (GameEvents.Instance == null) return;` as the first line, before the event subscriptions. This matches the pattern used by HappinessManager, CreditHUD, and CitizenInfoPanel.
2. In `UnsubscribeEvents()` (line 181): Add `if (GameEvents.Instance == null) return;` as the first line, before the event unsubscriptions.

Result:
```csharp
protected override void SubscribeEvents()
{
    if (GameEvents.Instance == null) return;
    GameEvents.Instance.WishGenerated += OnWishGenerated;
    GameEvents.Instance.WishFulfilled += OnWishFulfilled;
    GameEvents.Instance.RoomPlaced += OnRoomPlaced;
    GameEvents.Instance.RoomDemolished += OnRoomDemolished;
}

protected override void UnsubscribeEvents()
{
    if (GameEvents.Instance == null) return;
    GameEvents.Instance.WishGenerated -= OnWishGenerated;
    GameEvents.Instance.WishFulfilled -= OnWishFulfilled;
    GameEvents.Instance.RoomPlaced -= OnRoomPlaced;
    GameEvents.Instance.RoomDemolished -= OnRoomDemolished;
}
```

This is currently safe due to Autoload ordering (GameEvents initializes in _EnterTree before WishBoard's _Ready), but adding the guard makes the code consistent with every other subscriber in the codebase and prevents NullReferenceException if initialization order ever changes.
