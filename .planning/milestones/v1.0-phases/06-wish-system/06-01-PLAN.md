  <action>
    Create WishBoard.cs in Scripts/Autoloads/ following the SafeNode Autoload singleton pattern (same as CitizenManager, EconomyManager). WishBoard extends SafeNode.

    **WishBoard responsibilities:**
    1. Track active wishes: `Dictionary<string, WishTemplate> _activeWishes` keyed by citizen name
    2. Track placed room types: `Dictionary<string, int> _placedRoomTypes` counting instances of each room type on the ring (needed because RoomDemolished only provides segmentIndex, not room type -- see Pitfall 6 in research)
    3. Track segment-to-roomId mapping: `Dictionary<int, string> _segmentRoomIds` to look up room type on demolish
    4. Load all WishTemplate .tres resources from Resources/Wishes/ directory at startup
    5. Provide query API for external consumers

    **Namespace:** `OrbitalRings.Autoloads`

    **Required using statements:**
    ```csharp
    using System.Collections.Generic;
    using System.Linq;
    using Godot;
    using OrbitalRings.Citizens;
    using OrbitalRings.Core;
    using OrbitalRings.Data;
    ```

    **_Ready() implementation:**
    1. Set `Instance = this`
    2. Call `base._Ready()` (SafeNode lifecycle)
    3. Load all WishTemplate .tres files from `res://Resources/Wishes/` using `DirAccess.Open` + loop. For each .tres file, `ResourceLoader.Load<WishTemplate>(path)`. Store in `List<WishTemplate> _allTemplates`.
    4. Group templates by category for balanced wish generation: `Dictionary<WishTemplate.WishCategory, List<WishTemplate>> _templatesByCategory`
    5. Initialize `_placedRoomTypes` by scanning existing placed rooms via BuildManager (handle case where rooms are already placed at game start)
    6. Log loaded template count: `GD.Print($"WishBoard: Loaded {_allTemplates.Count} wish templates");`
    7. If zero templates loaded, `GD.PushWarning("WishBoard: No wish templates found in Resources/Wishes/")`.

    **SubscribeEvents() override:**
    ```csharp
    GameEvents.Instance.WishGenerated += OnWishGenerated;
    GameEvents.Instance.WishFulfilled += OnWishFulfilled;
    GameEvents.Instance.RoomPlaced += OnRoomPlaced;
    GameEvents.Instance.RoomDemolished += OnRoomDemolished;
    ```

    **UnsubscribeEvents() override:** Mirror all subscriptions with `-=`.

    **Event handlers:**
    - `OnWishGenerated(string citizenName, string wishId)`: Find WishTemplate by WishId from `_allTemplates`, add to `_activeWishes[citizenName]`.
    - `OnWishFulfilled(string citizenName, string wishId)`: Remove from `_activeWishes`.
    - `OnRoomPlaced(string roomType, int segmentIndex)`: Increment `_placedRoomTypes[roomType]`, store `_segmentRoomIds[segmentIndex] = roomType`. Then call `NudgeCitizensForRoom(roomType)` to reset visit timers on citizens whose active wish matches.
    - `OnRoomDemolished(int segmentIndex)`: Look up room type from `_segmentRoomIds[segmentIndex]`, decrement `_placedRoomTypes` count (remove key if zero), remove from `_segmentRoomIds`.

    **NudgeCitizensForRoom(string roomType):**
    Iterate `CitizenManager.Instance.Citizens`. For each citizen, check if their active wish (via `_activeWishes`) has a FulfillingRoomIds containing `roomType`. If so, call `citizen.NudgeVisit()` (this method will be added in Plan 02 -- for now, WishBoard.cs can reference it; the project compiles all .cs together).

    Actually, since Plan 02 adds NudgeVisit() to CitizenNode, and this plan runs first, do NOT call citizen.NudgeVisit() directly. Instead, create a public method `NudgeCitizensForRoom(string roomType)` that stores the roomType and sets a flag. Plan 02 will wire this.

    Better approach: WishBoard stores the nudge logic but guards with a null check. The method iterates `CitizenManager.Instance?.Citizens` and for each citizen whose name appears in `_activeWishes` with a matching FulfillingRoomIds entry, calls a method. Since NudgeVisit() won't exist yet, use reflection or simply emit a new lightweight event.

    SIMPLEST approach: Add a new event to WishBoard itself:
    ```csharp
    public event Action<string> WishNudgeRequested;  // citizenName
    ```
    In `OnRoomPlaced`, after tracking, iterate `_activeWishes` and for each entry whose WishTemplate.FulfillingRoomIds contains the placed roomType, invoke `WishNudgeRequested?.Invoke(citizenName)`. Plan 02 will have CitizenNode subscribe to this event.

    **Query API (public methods):**
    - `GetWishForCitizen(string citizenName)` -> `WishTemplate` or null (from `_activeWishes`)
    - `GetActiveWishCount()` -> `_activeWishes.Count`
    - `GetActiveWishes()` -> `IReadOnlyDictionary<string, WishTemplate>` (return `_activeWishes`)
    - `IsRoomTypeAvailable(string roomId)` -> bool (check `_placedRoomTypes.ContainsKey(roomId) && _placedRoomTypes[roomId] > 0`)
    - `GetRandomTemplate()` -> `WishTemplate` (picks random template from `_allTemplates` using `GD.Randi()`)
    - `GetRandomTemplateForCategory(WishCategory category)` -> `WishTemplate` (picks from `_templatesByCategory[category]`)

    **Register in project.godot:**
    Add WishBoard to the [autoload] section AFTER CitizenManager (WishBoard depends on CitizenManager.Instance for nudge):
    ```
    WishBoard="*res://Scripts/Autoloads/WishBoard.cs"
    ```
