## Unsatisfied Requirements

### ECON-03: Citizens assigned to work rooms generate bonus credits

**Requirement:** "Citizens assigned to work rooms generate bonus credits"
**Assigned Phase:** Phase 3 (formula), Phase 5 (events)
**REQUIREMENTS.md Status:** [x] Complete (incorrectly marked)
**Phase 3 VERIFICATION:** SATISFIED — notes `SetWorkingCitizenCount()` stub ready for Phase 5
**Phase 5 VERIFICATION:** Not covered (Phase 5 verifies CTZN-01..04 only)

**Root Cause:** Cross-phase integration gap. The infrastructure exists on both sides but was never connected:

- `EconomyManager.SetWorkingCitizenCount(int count)` exists (Phase 3, line 239)
- `_workingCitizenCount * Config.WorkBonusMultiplier` is used in `CalculateTickIncome()` (Phase 3, line 111)
- `GameEvents.CitizenEnteredRoom` / `CitizenExitedRoom` are emitted by `CitizenNode` (Phase 5, lines 480, 491)
- **No code subscribes to these events.** `_workingCitizenCount` stays at 0 for the entire session.

**Impact:** Work rooms placed by the player generate zero bonus credits. The income tooltip will always show "Work bonus: +0" regardless of how many citizens visit Work category rooms.

**Fix complexity:** Low. Add a subscriber (in CitizenManager or HappinessManager) that listens to CitizenEnteredRoom/CitizenExitedRoom, checks if the room category is Work, maintains a running count, and calls `EconomyManager.Instance.SetWorkingCitizenCount(count)`.
