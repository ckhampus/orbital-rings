**INTG-04 specifics:**
- AssignmentDistributesEvenly: 2 rooms at anchor 0 (cap 2) and anchor 5 (cap 2). 3 citizens: "Alice", "Bob", "Carol". Assert Math.Abs(room0 - room5) <= 1. Assert all 3 have non-null GetHomeForCitizen. Assert assigned.Count == 3.
- AssignmentSingleRoomGetsAll: 1 room at anchor 0 (cap 4). 3 citizens. Assert GetOccupantCount(0) == 3. Assert TotalHoused == 3.
- AssignmentDifferentCapacityRooms: room at anchor 0 (cap 2, simulating 1-seg), room at anchor 5 (cap 4, simulating 3-seg). 5 citizens. Assert no room exceeds its capacity: GetOccupantCount(0) <= 2 and GetOccupantCount(5) <= 4. Assert TotalHoused == 5.
- AssignmentFullRoomsLeaveCitizenUnhoused: 2 rooms at anchor 0 and anchor 5 (cap 2 each). 5 citizens: "Alice", "Bob", "Carol", "Dave", "Eve". Assert TotalHoused == 4. Assert exactly one citizen has null GetHomeForCitizen.

**INTG-05 specifics:**
- DemolishReassignsCitizens: 2 rooms at anchor 0 (cap 3) and anchor 5 (cap 3). Arrive 3 citizens. Subscribe to CitizenUnhoused and CitizenAssignedHome AFTER initial assignment. EmitRoomDemolished(0). Assert: citizens from room 0 are now in room 5. TotalHoused == 3 (no one lost). Note: after demolish, displaced count comes from room 0's occupancy before demolish. Use GetOccupantCount(0) before demolish to know expected displacement count.
- DemolishOnlyRoomLeavesAllUnhoused: 1 room at anchor 0 (cap 3). Arrive 2 citizens. Subscribe to CitizenUnhoused AFTER arrival. EmitRoomDemolished(0). Assert: CitizenUnhoused fires for each citizen. GetHomeForCitizen returns null for both. TotalHoused == 0.

**INTG-06 specifics:**
- WishFulfillmentCrossesTierAndUpdatesEconomy: Set EconomyManager.Instance.SetCitizenCount(5) and SetWorkingCitizenCount(0). Record quietIncome = CalculateTickIncome(). Pump 2 EmitWishFulfilled("Alice", "comfort"). Assert CurrentTier == MoodTier.Cozy. Record cozyIncome = CalculateTickIncome(). Assert cozyIncome > quietIncome. Specific values: quietIncome.ShouldBe(5), cozyIncome.ShouldBe(6).
- MoodTierChangedEventHasCorrectTiers: Subscribe to MoodTierChanged. Pump 2 EmitWishFulfilled. Assert tierChanges.Count == 1. Assert tierChanges[0] == (MoodTier.Cozy, MoodTier.Quiet).

**Design constraints (from CONTEXT.md locked decisions):**
- Use inline lambda event subscribers, NOT a reusable event recorder helper
- Behavior-focused method names (AssignmentDistributesEvenly, not TestAssignment)
- Shouldly assertions: .ShouldBe(), .ShouldBeNull(), .ShouldNotBeNull(), .ShouldBeGreaterThan(), .ShouldBeLessThanOrEqualTo()
- No #region directives
- Comment-header sections: `// --- INTG-04: Housing Assignment ---` etc.

**Design constraints (from research pitfalls):**
- Do NOT assert exact room assignments when ties are possible (FindBestRoom uses GD.Randi() tiebreak). Assert distribution properties (occupancy spread <= 1) instead.
- Do NOT use float assertions on mood values. Assert tier transitions (integer enum comparison) and income integers.
- For INTG-05 DemolishReassignsCitizens, note that OnRoomDemolished emits CitizenUnhoused THEN immediately tries FindBestRoom for reassignment. So CitizenUnhoused fires even when citizen gets reassigned to another room.
