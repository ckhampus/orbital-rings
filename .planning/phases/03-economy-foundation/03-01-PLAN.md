Create a markdown file with the economy balance spreadsheet. This must be done BEFORE any code changes. The spreadsheet must model:

**Income Table (per tick, tick interval = 5.5 seconds):**
| Citizens | Base Station Income | Citizen Income (sqrt scaling) | Work Bonus (example: 2 workers) | Happiness Mult (at various levels) | Total Per Tick |
At milestones: 0, 5, 15, 30 citizens.

Formula: `tickIncome = roundToInt((baseStationIncome + perCitizen * sqrt(citizenCount) + workBonus) * happinessMultiplier)`
- BaseStationIncome = 1 (trickle at zero citizens per user decision)
- PassiveIncomePerCitizen = 2.0 (per-sqrt-citizen coefficient)
- WorkBonusMultiplier = 1.25 (small bonus per worker, down from 1.5 per user decision)
- HappinessMultiplierCap = 1.3 (subtle, down from 2.0 per user decision)
- Happiness multiplier formula: `1.0 + (happiness * (cap - 1.0))` where happiness is 0.0-1.0

**Cost Table:**
| Room Category | 1-seg Inner | 1-seg Outer | 2-seg Inner | 2-seg Outer | 3-seg Inner | 3-seg Outer |
For all 5 categories (Housing, LifeSupport, Work, Comfort, Utility).

Formula: `cost = roundToInt(baseCost * categoryMult * segments * (sizeDiscount ^ (segments-1)) * rowMult)`
- BaseRoomCost = 100
- SizeDiscountFactor = 0.92 (down from 0.85 per user decision, gives ~5-8% per additional segment)
- OuterRowCostMultiplier = 1.1 (outer segments are physically larger)
- Category multipliers: Housing=0.7, LifeSupport=0.85, Work=1.0, Comfort=1.15, Utility=1.3

**Starting Credits Validation:**
- StartingCredits must afford 5-6 cheap 1-segment Housing/LifeSupport rooms on inner row
- Cheapest room: 1-seg Housing inner = 100 * 0.7 * 1 * 1.0 * 1.0 = 70
- 5 rooms at 70 = 350, 6 rooms at 70 = 420
- LifeSupport 1-seg inner = 100 * 0.85 * 1 * 1.0 * 1.0 = 85
- Mix of 3 Housing + 3 LifeSupport = 3*70 + 3*85 = 465
- Recommend StartingCredits = 750 (affords ~5-6 cheap rooms with budget for one medium room)

**Accumulation Curves:**
Show how long (in ticks and real seconds) it takes to afford rooms at each citizen milestone:
- At 0 citizens: income = 1/tick, Housing room (70) takes 70 ticks = 385 seconds (~6.4 min)
- At 5 citizens: income = ~6/tick, Housing room takes ~12 ticks = 66 seconds
- Etc.

**Runaway Validation:**
- Verify 30-citizen income is NOT more than 10x the 5-citizen income (the research warning sign)
- Verify happiness multiplier cap (1.3x) keeps income bounded
- Show that diminishing returns via sqrt flatten the curve appropriately

The spreadsheet is a planning artifact, NOT code. Write it as readable markdown tables with clear commentary.

```csharp
/// <param name="amount">Credits earned from this income tick.</param>
public event Action<int> IncomeTicked;

/// <param name="amount">Credits spent (positive number).</param>
public event Action<int> CreditsSpent;

/// <param name="amount">Credits refunded (positive number).</param>
public event Action<int> CreditsRefunded;

public void EmitIncomeTicked(int amount) => IncomeTicked?.Invoke(amount);
public void EmitCreditsSpent(int amount) => CreditsSpent?.Invoke(amount);
public void EmitCreditsRefunded(int amount) => CreditsRefunded?.Invoke(amount);
```

These events are separate from CreditsChanged. CreditsChanged carries the new balance total. The new events carry the delta amount for the floating "+N"/"-N" HUD display.
