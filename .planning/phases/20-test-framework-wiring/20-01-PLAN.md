  <name>Task 1: Add nuget.org package source and configure conditional test compilation</name>
  <files>NuGet.Config, Orbital Rings.csproj, Tests/.gdignore</files>
  <action>
1. **NuGet.Config** -- Add nuget.org as a second package source alongside the existing godot-local source:
   ```xml
   <add key="nuget.org" value="https://api.nuget.org/v3/index.json" />
   ```
   Place it AFTER the existing godot-local entry. Do not remove or modify the godot-local source.

2. **Orbital Rings.csproj** -- Add the Chickensoft conditional test compilation pattern. The existing PropertyGroup and its contents must be preserved. Add the following AFTER the existing PropertyGroup:

   a. A new PropertyGroup that defaults `RunTests` to false:
      ```xml
      <PropertyGroup>
        <RunTests>false</RunTests>
      </PropertyGroup>
      ```

   b. A conditional PropertyGroup that enables tests in Debug/ExportDebug (unless SkipTests is set):
      ```xml
      <PropertyGroup Condition="('$(Configuration)' == 'Debug' or '$(Configuration)' == 'ExportDebug') and '$(SkipTests)' != 'true'">
        <RunTests>true</RunTests>
        <DefineConstants>$(DefineConstants);RUN_TESTS</DefineConstants>
      </PropertyGroup>
      ```

   c. A conditional ItemGroup for test packages (only when RunTests is true):
      ```xml
      <ItemGroup Condition="'$(RunTests)' == 'true'">
        <PackageReference Include="Chickensoft.GoDotTest" Version="2.0.30" />
        <PackageReference Include="Chickensoft.GodotTestDriver" Version="3.1.62" />
        <PackageReference Include="Shouldly" Version="4.3.0" />
        <PackageReference Include="LightMoq" Version="0.1.0" />
        <PackageReference Include="LightMock.Generator" Version="1.2.3" />
      </ItemGroup>
      ```
      NOTE: If `LightMoq` version 0.1.0 is not found on nuget.org, try `Chickensoft.LightMoq`. If `LightMock.Generator` is pulled transitively by LightMoq, the explicit reference is harmless -- keep it.

   d. A conditional ItemGroup that excludes test files from non-test builds:
      ```xml
      <ItemGroup Condition="'$(RunTests)' != 'true'">
        <Compile Remove="Tests/**/*.cs" />
        <None Remove="Tests/**/*" />
        <EmbeddedResource Remove="Tests/**/*" />
      </ItemGroup>
      ```

3. **Tests/.gdignore** -- Create the `Tests/` directory and place an empty `.gdignore` file in it. This prevents Godot from importing .tscn/.tres files from the Tests directory into export builds. The file content is empty (just the file's existence matters).

4. Run `dotnet restore` to verify NuGet pulls all five test packages without errors.

5. Run `dotnet build` to verify the project compiles with the new conditional compilation setup. The RUN_TESTS define should be active in Debug configuration.

6. Run `dotnet format` to ensure consistent code style per CLAUDE.md.

IMPORTANT: Do NOT modify the existing PropertyGroup (TargetFramework, EnableDynamicLoading, RootNamespace). Only ADD new PropertyGroups and ItemGroups after it.
