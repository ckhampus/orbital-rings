Add a public `Reset()` method to each of the 7 singletons. Each Reset() clears ALL mutable state and stops owned timers, returning the singleton to a "just loaded, no game data" state. Reset() must NOT touch the `Instance` reference, `[Export] Config` resources, or read-only caches loaded in `_Ready()`.

Use the complete mutable field catalog from 21-RESEARCH.md (section "Complete Mutable Field Catalog") as the authoritative reference. The exact fields for each singleton:

**EconomyManager.Reset():**
- `_credits = 0` (not Config.StartingCredits -- "no game data" state)
- `_citizenCount = 0`
- `_workingCitizenCount = 0`
- `_workingCitizens.Clear()`
- `_currentTierMultiplier = 1.0f`
- `_incomeTimer?.Stop()`
- `StateLoaded = false`

**HousingManager.Reset():**
- `_housingRoomCapacities.Clear()`
- `_roomOccupants.Clear()`
- `_citizenHomes.Clear()`
- `_isRestoring = false`
- `_onRoomPlaced = null`
- `_onRoomDemolished = null`
- `_onCitizenArrived = null`
- `StateLoaded = false`

**HappinessManager.Reset():**
- `_lifetimeHappiness = 0`
- `_moodSystem = new MoodSystem(Config ?? new HappinessConfig())` (recreate fresh, don't try to reset internals)
- `_lastReportedTier = MoodTier.Quiet`
- `_unlockedRooms.Clear()` then `_unlockedRooms.UnionWith(new[] { "bunk_pod", "air_recycler", "workshop", "reading_nook", "storage_bay", "garden_nook" })` (reset to starter rooms)
- `_crossedMilestoneCount = 0`
- `_arrivalTimer?.Stop()`
- Leave `_arrivalCanvasLayer` untouched (UI node, no game state)

**SaveManager.Reset():**
- `PendingLoad = null`
- `_saving = false`
- `_pendingLoadFrames = -1`
- `_debounceTimer?.Stop()`
- Null all delegate reference fields: `_onRoomPlaced`, `_onRoomDemolished`, `_onWishFulfilled`, `_onCitizenArrived`, `_onMoodTierChanged`, `_onWishCountChanged`, `_onCreditsChanged`, `_onBlueprintUnlocked`, `_onCitizenAssignedHome`, `_onCitizenUnhoused`

**BuildManager.Reset():**
- `_mode = BuildMode.Normal`
- `_selectedRoom = null`
- `_anchorFlatIndex = -1`
- `_currentSize = 1`
- `_startPos = 0`
- `_pendingDemolishIndex = -1`
- `_placedRooms.Clear()`
- If `_ghostMesh != null && Node.IsInstanceValid(_ghostMesh)`: `_ghostMesh.QueueFree()` then `_ghostMesh = null`
- Do NOT clear `_roomDefinitions` (read-only cache) or touch `_ringVisual` (scene tree reference)

**CitizenManager.Reset():**
- `_citizens.Clear()` (do NOT QueueFree citizen nodes -- they may not exist in test context)
- `_selectedCitizen = null`
- `_grid = null`
- `StateLoaded = false`
- Leave `_camera`, `_ringPlane`, `_infoPanel` untouched (UI/scene references)

**WishBoard.Reset():**
- `_activeWishes.Clear()`
- `_placedRoomTypes.Clear()`
- `_segmentRoomIds.Clear()`
- `WishNudgeRequested = null` (WishBoard-owned event, not a GameEvents event)
- Do NOT clear `_allTemplates` or `_templatesByCategory` (read-only caches loaded in _Ready())

IMPORTANT: Before writing each Reset(), read the full singleton source file to verify the field names match current code. The RESEARCH.md catalog was based on a point-in-time snapshot -- field names or types may have been renamed. Cross-reference carefully.
