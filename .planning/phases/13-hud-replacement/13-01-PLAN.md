Create `Scripts/UI/MoodHUD.cs` as a `MarginContainer` subclass in `OrbitalRings.UI` namespace, following the CreditHUD/PopulationDisplay programmatic pattern exactly. The widget contains both the wish counter and tier label in a single HBoxContainer.

**Structure (all built programmatically in _Ready):**
- `HBoxContainer _hbox` (separation=6, MouseFilter.Ignore)
- `Label _heartLabel`: text="\u2665" (Unicode heart), font_color=coral (0.95, 0.55, 0.55), font_size=20, MouseFilter.Ignore
- `Label _countLabel`: font_color=warm white (0.95, 0.93, 0.90), font_size=20, MouseFilter.Ignore
- `Label _tierLabel`: font_size=20, MouseFilter.Ignore (color set from tier lookup)

**Tier color dictionary** (warm spectrum, cool-to-warm):
- Quiet: soft grey-blue (0.65, 0.72, 0.78)
- Cozy: warm tan/sand (0.85, 0.75, 0.55)
- Lively: warm amber (0.95, 0.72, 0.42)
- Vibrant: warm coral-orange (0.95, 0.60, 0.40)
- Radiant: bright gold (1.00, 0.85, 0.35)

**Event subscriptions (in _Ready, unsubscribe in _ExitTree):**
- `GameEvents.Instance.WishCountChanged += OnWishCountChanged`
- `GameEvents.Instance.MoodTierChanged += OnMoodTierChanged`

**Initial state (in _Ready, AFTER building UI, no animation):**
- `_countLabel.Text = (HappinessManager.Instance?.LifetimeWishes ?? 0).ToString()`
- Read `HappinessManager.Instance?.CurrentTier ?? MoodTier.Quiet`, set `_tierLabel.Text` to tier name via `.ToString()`, set font_color from tier color dictionary (no cross-fade, just set directly)

**OnWishCountChanged(int newCount):**
- Set `_countLabel.Text = newCount.ToString()`
- Kill-before-create `_pulseTween`: set `_countLabel.Scale = new Vector2(1.2f, 1.2f)`, tween to (1.0, 1.0) over 0.3s with EaseType.Out + TransitionType.Elastic (exact PopulationDisplay pattern)

**OnMoodTierChanged(MoodTier newTier, MoodTier previousTier):**
1. Set `_tierLabel.Text = newTier.ToString()`
2. Kill-before-create `_tierColorTween`: use `TweenMethod` with `Callable.From<Color>(c => _tierLabel.AddThemeColorOverride("font_color", c))` to cross-fade from `TierColors[previousTier]` to `TierColors[newTier]` over 0.3s with EaseType.Out. Use `CreateTween()` on the MoodHUD node (not _tierLabel) because TweenMethod operates on an arbitrary callable, not a node property.
3. Kill-before-create `_tierPopTween`: set `_tierLabel.Scale = new Vector2(1.15f, 1.15f)`, tween to (1.0, 1.0) over 0.3s with EaseType.Out + TransitionType.Elastic (slightly smaller than wish pulse for subtlety)
4. Call `SpawnTierNotification(newTier)` to show floating text

**SpawnTierNotification(MoodTier tier):**
- Create `new FloatingText()`, `AddChild(floater)`
- Position: `new Vector2(_tierLabel.Position.X, -20)` (above the tier label)
- Call `floater.Setup($"Station mood: {tier}", TierColors[tier], startPos)`

**Tween state fields:** `_pulseTween`, `_tierColorTween`, `_tierPopTween` (three separate Tween references for independent kill-before-create)

**Required usings:** `Godot`, `System.Collections.Generic`, `OrbitalRings.Autoloads`, `OrbitalRings.Data`
