**GameEvents.cs** (`Scripts/Autoloads/GameEvents.cs`):
- Namespace: `OrbitalRings.Autoloads`
- `public partial class GameEvents : Node`
- Static `Instance` property set in `_Ready()`
- Use pure C# `event Action<T>` delegates (NOT Godot [Signal]) per research recommendation ‚Äî avoids marshalling overhead and IsConnected bugs (GitHub #76690)
- Declare event stubs for ALL future phases in one place:
  - Camera: `CameraOrbitStarted`, `CameraOrbitStopped`
  - Room (Phase 4): `RoomPlaced(string roomType, int segmentIndex)`, `RoomDemolished(int segmentIndex)`
  - Citizen (Phase 5): `CitizenArrived(string citizenName)`, `CitizenDeparted(string citizenName)`
  - Wish (Phase 6): `WishGenerated(string citizenName, string wishType)`, `WishFulfilled(string citizenName, string wishType)`
  - Economy (Phase 3): `CreditsChanged(int newBalance)`, `HappinessChanged(float newHappiness)`
  - Progression (Phase 7): `BlueprintUnlocked(string roomType)`
- Add null-safe `Emit*()` helper methods for each event using `?.Invoke()` pattern
- 2-space indentation per .editorconfig

**SafeNode.cs** (`Scripts/Core/SafeNode.cs`):
- Namespace: `OrbitalRings.Core`
- `public partial class SafeNode : Node`
- Override `_EnterTree()` ‚Üí calls `SubscribeEvents()`
- Override `_ExitTree()` ‚Üí calls `UnsubscribeEvents()`
- Both methods are `protected virtual` with empty default implementations
- XML doc comments explaining the convention: subclasses override both, every `+=` must have a matching `-=`
- Note in comments: pure C# `-=` on never-connected delegate is safe no-op (unlike Godot's Disconnect which throws)

**project.godot**: Add `[autoload]` section registering GameEvents:
```
[autoload]
GameEvents="*res://Scripts/Autoloads/GameEvents.cs"
```
The `*` prefix means enabled. GameEvents must be the first (and currently only) Autoload.
