    In `_Ready()`, after existing setup:
    ```csharp

    ```
    Add method:
    ```csharp
    private void FindBuildManager()
    {
        _buildManager = GetNode<OrbitalRings.Build.BuildManager>("/root/BuildManager");
    }
    ```
    Or simply access via `BuildManager.Instance` in the click handlers (since it's a singleton, and it's guaranteed to be initialized before any user interaction occurs).

    **2. Modify left-click handler in `_Input`:**
    The existing code:
    ```csharp
    if (_hoveredFlatIndex >= 0)
    {
        SelectSegment(_hoveredFlatIndex);
        GetViewport().SetInputAsHandled();
    }
    ```

    Replace with build-mode-aware delegation:
    ```csharp
    if (_hoveredFlatIndex >= 0)
    {
        // Check if UI is consuming this click (BuildPanel visible and mouse is over it)
        if (IsMouseOverUi())
        {
            return; // Let UI handle it
        }

        var buildMgr = OrbitalRings.Build.BuildManager.Instance;
        if (buildMgr != null && buildMgr.CurrentMode != BuildMode.Normal)
        {
            // Delegate to BuildManager in build modes
            buildMgr.OnSegmentClicked(_hoveredFlatIndex);
            GetViewport().SetInputAsHandled();
        }
        else
        {
            // Normal selection behavior
            SelectSegment(_hoveredFlatIndex);
            GetViewport().SetInputAsHandled();
        }
    }
    ```

    **3. Add UI overlap check:**
    ```csharp
    private bool IsMouseOverUi()
    {
        // If any Control node has focus or the mouse is over a UI element,
        // don't process as a 3D click
        var focused = GetViewport().GuiGetFocusOwner();
        if (focused != null) return true;

        // More robust: check if mouse is in the lower portion where BuildPanel lives
        // This is a simplified check; the real guard is MouseFilter.Stop on BuildPanel
        return false;
    }
    ```
    Note: The primary guard is MouseFilter.Stop on BuildPanel which prevents _Input from firing on SegmentInteraction for clicks on the UI. The IsMouseOverUi check is a safety fallback.

    **4. Modify hover behavior for build mode:**
    In `UpdateHover()`, after computing the flatIndex (step 7 — "Update hover state if changed"):
    ```csharp
    // After existing hover visual update...
    // Delegate hover to BuildManager for drag-to-resize in Placing mode
    var buildMgr = OrbitalRings.Build.BuildManager.Instance;
    if (buildMgr != null && buildMgr.CurrentMode == BuildMode.Placing)
    {
        buildMgr.OnSegmentHovered(flatIndex);
    }
    ```

    **5. Modify Escape handling:**
    In the Escape key handler, check build mode first:
    ```csharp
    else if (@event is InputEventKey key && key.Pressed && key.Keycode == Key.Escape)
    {
        var buildMgr = OrbitalRings.Build.BuildManager.Instance;
        if (buildMgr != null && buildMgr.CurrentMode != BuildMode.Normal)
        {
            // Build mode handles Escape (cancel placement or exit build mode)
            // BuildManager._Input will handle this, or call ExitBuildMode here
            buildMgr.ExitBuildMode();
            GetViewport().SetInputAsHandled();
        }
        else
        {
            DeselectSegment();
        }
    }
    ```

    **6. Suppress normal hover highlighting in build modes:**
    When BuildManager is in Placing or Demolish mode, the normal hover/selection highlighting should be modified:
    - In Placing mode: don't show normal hover highlight (BuildManager's ghost preview provides visual feedback)
    - In Demolish mode: still show hover highlight (helps identify which room will be demolished)
    - Add a check at the top of `UpdateHover()`: if in Placing mode, still compute _hoveredFlatIndex for BuildManager delegation but skip the normal `SetSegmentState(Hovered)` call.

    **IMPORTANT: Do NOT break existing Normal mode behavior.** The segment selection, hover, and tooltip should work exactly as before when BuildManager is in Normal mode. All changes are additive — check build mode before altering behavior.

    Add `using OrbitalRings.Autoloads;` at the top if not already present (for BuildMode enum defined in GameEvents.cs).
