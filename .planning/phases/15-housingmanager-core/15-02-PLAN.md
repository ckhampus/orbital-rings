Three changes to SaveManager.cs:

**1. CollectGameState -- Write HomeSegmentIndex (around line 316-329):**

In the citizen serialization loop, add HomeSegmentIndex to the SavedCitizen initializer:

```csharp
data.Citizens.Add(new SavedCitizen
{
    // ... all existing fields unchanged ...
    HomeSegmentIndex = HousingManager.Instance?.GetHomeForCitizen(citizenData.CitizenName)
});
```

This writes the anchor index (or null for unhoused) into the save file. The `int?` serializes to JSON as either a number or `null`.

**2. ApplySceneState -- Restore housing assignments (after the citizen restoration block, around line 455):**

After the existing citizen restoration loop and before the wish board restoration, add a housing restoration step:

```csharp
// Restore housing assignments (must come after rooms AND citizens are restored)
if (HousingManager.Instance != null)
{
    var assignments = data.Citizens
        .Select(c => (c.Name, c.HomeSegmentIndex))
        .ToList();
    HousingManager.Instance.RestoreFromSave(assignments);
}
```

This MUST come after both rooms and citizens are restored because:
- HousingManager needs rooms in its capacity dictionary (populated via RoomPlaced events during RestorePlacedRoom)
- HousingManager needs CitizenNode references to set HomeSegmentIndex properties

RestoreFromSave handles stale references internally (demolished rooms become unhoused, then reassigned). It uses the `_isRestoring` flag to suppress event emission, preventing autosave loops (see Pitfall 3 in RESEARCH.md).

**3. Housing event autosave subscriptions:**

Add two new delegate fields alongside the existing ones (around line 123-130):

```csharp
private Action<string, int> _onCitizenAssignedHome;
private Action<string> _onCitizenUnhoused;
```

In SubscribeEvents (around line 191-203), add:

```csharp
_onCitizenAssignedHome = (_, _) => OnAnyStateChanged();
_onCitizenUnhoused = _ => OnAnyStateChanged();
GameEvents.Instance.CitizenAssignedHome += _onCitizenAssignedHome;
GameEvents.Instance.CitizenUnhoused += _onCitizenUnhoused;
```

In UnsubscribeEvents (around line 205-217), add:

```csharp
GameEvents.Instance.CitizenAssignedHome -= _onCitizenAssignedHome;
GameEvents.Instance.CitizenUnhoused -= _onCitizenUnhoused;
```

This ensures housing changes trigger autosave through the existing debounce timer, using the same pattern as RoomPlaced/CitizenArrived (same pattern per user decision in CONTEXT.md).

**Important:** The `using System.Linq` import already exists in SaveManager.cs (line 3), so the `.Select().ToList()` LINQ calls compile without additional imports.
