**SaveManager class:**
- Namespace: `OrbitalRings.Autoloads`
- Extends `Node` (not SafeNode -- subscribes to events directly in _Ready/_ExitTree)
- `public static SaveManager Instance { get; private set; }`
- Private const `SavePath = "user://save.json"`
- Private `Timer _debounceTimer` (0.5s, one-shot)
- Private `bool _saving` flag to prevent re-entrant saves

**_Ready():**
1. Set Instance = this
2. Create debounce Timer (0.5s, OneShot=true), add as child, connect Timeout to PerformSave
3. Subscribe to ALL state-change events on GameEvents.Instance:
   - RoomPlaced, RoomDemolished (Action<string, int> and Action<int>)
   - WishFulfilled (Action<string, string>)
   - CitizenArrived (Action<string>)
   - HappinessChanged (Action<float>)
   - CreditsChanged (Action<int>)
   - BlueprintUnlocked (Action<string>)
   All handlers call `OnAnyStateChanged()`

**_ExitTree():** Unsubscribe all event handlers (mirror SubscribeEvents).

**OnAnyStateChanged():** Reset debounce timer: `_debounceTimer.Stop(); _debounceTimer.Start();`

**PerformSave():**
1. Guard: if `_saving` return (prevent re-entrancy)
2. Set `_saving = true`
3. Call `CollectGameState()` to build SaveData
4. Serialize with `System.Text.Json.JsonSerializer.Serialize(saveData, new JsonSerializerOptions { WriteIndented = true })`
5. Open `SavePath` with `using var file = FileAccess.Open(SavePath, FileAccess.ModeFlags.Write)`
6. Guard null file (log warning and return)
7. `file.StoreString(json)`
8. Set `_saving = false`

**CollectGameState() -> SaveData:**
- Credits from `EconomyManager.Instance.Credits`
- Happiness from `HappinessManager.Instance.Happiness`
- CrossedMilestoneCount from `HappinessManager.Instance.GetCrossedMilestoneCount()`
- HousingCapacity from `HappinessManager.Instance.GetHousingCapacity()`
- UnlockedRooms from `HappinessManager.Instance.GetUnlockedRoomIds().ToList()`
- PlacedRooms from `BuildManager.Instance.GetAllPlacedRooms()` mapped to SavedRoom objects
- Citizens from `CitizenManager.Instance.Citizens` mapped to SavedCitizen objects:
  - Snap visiting citizens to walkway: use citizen.CurrentAngle regardless of visit state
  - Store citizen.Data.PrimaryColor as R/G/B floats, same for SecondaryColor
  - Store citizen.CurrentWish?.WishId as CurrentWishId (null if no wish)
  - Store citizen.Direction
- ActiveWishes from `WishBoard.Instance.GetActiveWishes()` mapped to Dictionary<string, string> (citizenName -> wishId)

**Load() -> SaveData (nullable):**
1. Check `FileAccess.FileExists(SavePath)`, return null if not
2. Open with `using var file = FileAccess.Open(SavePath, FileAccess.ModeFlags.Read)`, guard null
3. Wrap deserialization in try/catch(JsonException): `JsonSerializer.Deserialize<SaveData>(file.GetAsText())`
4. On exception: `GD.PushWarning("SaveManager: Corrupted save file: {ex.Message}")`, return null
5. Return the deserialized SaveData

**ApplyState(SaveData data):**
This is called BEFORE ChangeSceneToFile. Sets StateLoaded flags and restores all singleton state.

1. Set `CitizenManager.StateLoaded = true`
2. Set `HappinessManager.StateLoaded = true`
3. Set `EconomyManager.StateLoaded = true`
4. `EconomyManager.Instance.RestoreCredits(data.Credits)`
5. `HappinessManager.Instance.RestoreState(data.Happiness, new HashSet<string>(data.UnlockedRooms), data.CrossedMilestoneCount, data.HousingCapacity)`

Note: Rooms, citizens, and wishes must be restored AFTER the game scene loads (managers need RingVisual, Grid, etc.). Add a public method `ApplySceneState(SaveData data)` that is called from the game scene's _Ready or via CallDeferred after scene change:

**ApplySceneState(SaveData data):**
1. Restore rooms: `BuildManager.Instance.ClearAllRooms()` then for each SavedRoom call `BuildManager.Instance.RestorePlacedRoom(...)`
2. Restore citizens: `CitizenManager.Instance.ClearCitizens()` then for each SavedCitizen call `CitizenManager.Instance.SpawnCitizenFromSave(...)`
3. Restore wish board: `WishBoard.Instance.RestoreActiveWishes(data.ActiveWishes)`
4. Store pending SaveData in a private field `_pendingSceneState` that ApplySceneState reads.

Actually, simpler approach: Store the SaveData in a public property `public SaveData PendingLoad { get; set; }`. After ChangeSceneToFile, the game scene's initialization code (or a deferred call from SaveManager) checks `PendingLoad` and calls `ApplySceneState`.

Use `GetTree().Connect("node_added", ...)` or a simpler pattern: in `_Process`, check if PendingLoad != null and if RingVisual/BuildManager are ready, then apply and clear. Or even simpler: use `CallDeferred` after scene change.

Best approach per research: Save manager subscribes to scene tree changed signal, and when the game scene root appears, calls ApplySceneState via CallDeferred (one frame delay so all _Ready methods complete first).

**ClearSave():** Delete the save file using `DirAccess.RemoveAbsolute(ProjectSettings.GlobalizePath(SavePath))` if it exists.

Actually simplest robust approach: Use `CallDeferred` from the title screen AFTER `ChangeSceneToFile`:
```csharp
// In title screen after scene change:
SaveManager.Instance.CallDeferred("ApplySceneStateDeferred");
```
But CallDeferred uses string method names in Godot C# which requires [Export] or specific binding.

Best approach: In SaveManager._Process, check if PendingLoad is set and BuildManager.Instance is available (non-null, which means its _Ready ran). Use a small frame delay:

```csharp
private int _pendingLoadFrames = -1;

public override void _Process(double delta)
{
    if (_pendingLoadFrames >= 0)
    {
        _pendingLoadFrames++;
        if (_pendingLoadFrames >= 2) // Wait 2 frames for all _Ready to complete
        {
            if (PendingLoad != null)
            {
                ApplySceneState(PendingLoad);
                PendingLoad = null;
            }
            _pendingLoadFrames = -1;
        }
    }
}

// Called from title screen after ChangeSceneToFile
public void ScheduleSceneRestore()
{
    _pendingLoadFrames = 0;
}
```

This is clean and reliable. The title screen calls `SaveManager.Instance.ScheduleSceneRestore()` right after `GetTree().ChangeSceneToFile(...)`.

```csharp
// where _onRoomPlaced is a field: private Action<string, int> _onRoomPlaced;

```
Store delegates as fields so they can be unsubscribed in _ExitTree.
